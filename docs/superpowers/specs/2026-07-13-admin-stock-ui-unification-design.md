# 管理后台库存页面 UI 统一设计

## 目标

将 `stock-adjustment-review.html`（库存调整审核）和 `inventory-movements.html`（库存流水）统一为与 `stock_summary.html`（库存管理）一致的桌面后台视觉体系。

本次只调整页面结构、信息层级和样式，不修改查询、审核、驳回、导出、数据库或 API 行为。

## 使用场景

- 使用者：管理后台操作人员。
- 主要设备：桌面电脑和宽屏浏览器。
- 不为手机阅读单独设计，不把宽表格改成卡片列表。
- 页面可以横向滚动，但核心操作区必须在常见桌面宽度内清晰可见。

## 统一视觉规则

参考 `stock_summary.html`：

- 页面背景：`#f4f2f5`。
- 主色：`#4A154B`。
- 页面容器最大宽度：`1280px`，居中显示。
- 内容区使用白色卡片、14px 圆角、淡紫色阴影。
- 标题使用紫色，顶部结构统一为标题说明区与右侧操作区。
- 普通按钮为白底深色文字；主操作按钮为紫底白字。
- 输入框、下拉框、日期框统一为 9px 圆角和浅紫边框。
- 表格使用独立白色卡片和滚动容器。
- 表头使用浅紫背景、紫色文字，并保持顶部粘性。
- 表格保留高信息密度，不为移动端折叠成卡片。
- 数量增加使用绿色，数量减少使用红色，普通数字使用紫色强调。

## 库存流水页面

### 顶部卡片

包含：

- 标题：库存流水。
- 说明：展示库存变化记录，可按日期、业务员和类型筛选。
- 返回管理看板按钮。
- 刷新按钮。

### 筛选区

筛选条件位于同一行，按以下顺序排列：

1. 开始日期；
2. 结束日期；
3. 业务员；
4. 库存变化类型；
5. 查询；
6. 导出 Excel。

查询按钮使用主按钮样式；导出按钮使用普通按钮样式。

### 状态信息

在筛选区下方显示：

- 正在查询；
- 查询失败；
- 当前结果条数。

状态文本不占用独立大卡片。

### 流水表格

- 保留现有全部字段和顺序。
- 表格最小宽度允许超过容器，通过横向滚动查看。
- 表头固定在滚动容器顶部。
- 数量变化列：正数绿色、负数红色、零值灰色。
- 前库存、后库存使用统一数字样式。
- 来源单号、条码和工号保持不换行，避免列宽失控。
- 空结果使用表格内居中的空状态行。

## 库存调整审核页面

### 顶部卡片

包含：

- 标题：库存调整审核。
- 说明：按提交时间从早到晚处理；库存为审核时实时库存。
- 返回管理看板按钮。
- 刷新按钮。

### 概览区

在队列上方显示两个指标：

- 待审核申请数；
- 待审核商品行数。

指标卡片沿用库存管理页面的 `metric-grid` 风格。

### 申请分组

每个申请使用独立白色面板，结构为：

1. 面板头部：申请单号、员工、提交时间；
2. 原因区：原因、说明和备注；
3. 商品明细表；
4. 右下角操作区。

申请之间保持明显间距，不使用简单分割线堆叠。

### 审核明细表

保留现有字段：

- 商品名称/规格口味；
- 条码；
- 当前库存；
- 调整；
- 审核后库存。

样式要求：

- 调整为正数时绿色；
- 调整为负数时红色；
- 审核后库存为负数时红色加粗；
- 条码不换行；
- 表头与库存管理页面一致。

### 操作区

- “同意”使用紫色主按钮。
- “驳回”使用白底红字或红色边框按钮。
- 操作区右对齐。
- 操作执行期间保持现有全队列按钮禁用逻辑。
- 不改变现有 `prompt` 驳回理由交互，本次不新增弹窗组件。

### 空状态与错误状态

- 无待审核申请时，在白色卡片中显示居中空状态。
- 加载失败时使用红色错误文字。
- 加载中使用灰色状态文字。

## 文件边界

计划修改：

- `stock-adjustment.css`
- `stock-adjustment-review.html`
- `stock-adjustment-review.js`
- `inventory-movements.html`
- `inventory-movements-page.js`
- `tests/stock-adjustment-pages.test.mjs`
- `docs/status/STOCK-ADJUSTMENT-PHASE-C.md`

不修改：

- 数据库 migration；
- `stock-adjustment-api.js` 的请求接口；
- 审核和流水的业务规则；
- 员工端库存调整页面；
- 整个管理后台的其他页面样式。

## 测试与验收

自动化检查需要验证：

- 两个页面都使用统一页面容器、顶部卡片、按钮和表格结构；
- 流水页面保留所有筛选控件和全部表头字段；
- 审核页面保留同意和驳回操作；
- 审核脚本输出概览数据和统一申请面板类名；
- 流水脚本为正负数量输出对应样式类；
- 现有 API 调用名称和参数顺序不变；
- JavaScript 语法检查通过；
- 全量 Node 测试无新增失败。

页面实际效果由用户在 `stock-adjustment-phase-c` 自动部署页面上验收。

## 非目标

- 不优化手机端阅读。
- 不改成响应式卡片列表。
- 不引入前端框架或新依赖。
- 不抽取全站 `admin.css`。
- 不改审核业务流程、驳回流程或导出格式。
- 不修改库存管理页面本身。
